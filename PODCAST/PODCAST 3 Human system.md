### **Host:**
สวัสดีครับ กลับมาพบกันครั้งนี้ เป็นเรื่องที่เราต้องทำความเข้าใจให้ดีเลยนะครับ
เพราะผมมองว่ามันเป็นกรอบคิด ในการที่คนจะใช้คอมพิวเตอร์ แก้ปัญหาให้ตัวเอง

ยิ่งเป็นระบบ IoT ที่มีเซนเซอร์ด้วยแล้ว มันทำงานคล้ายคนมากเลยครับ

ทั้งในการรับรู้ ตัดสินใจ  และสั่งการ

### **Guest:**

ดีครับ งั้นวันนี้เรามาเริ่ม “บทเปิด” ของตอนที่ 3 แบบค่อย ๆ ต่อกลอนกันเหมือนเดิมนะครับ เพราะสิ่งที่ Host พูดเมื่อกี้ — _“IoT ทำงานคล้ายคนมาก ทั้งการรับรู้ ตัดสินใจ และสั่งการ”_ — นั่นคือหัวใจของตอนนี้เลย

ผมอยากให้ทุกคนลองนึกภาพแบบง่ายที่สุดก่อนว่า **มนุษย์ = ระบบรับรู้ที่ซับซ้อนที่สุดในโลก** และทุกอย่างที่เราออกแบบในโลกของเซนเซอร์–คอมพิวเตอร์–IoT จริง ๆ แล้วมันคือ “การเลียนแบบมนุษย์” ทั้งนั้น

## **มนุษย์รับรู้โลกแบบต่อเนื่อง (Analog)**

ก่อนที่เราจะพูดถึง ADC หรือดิจิทัล ผมอยากให้ทุกคนย้อนกลับมาที่ตัวเองก่อน

มนุษย์รับรู้โลกแบบ **ต่อเนื่อง** ไม่มีการแบ่งเป็นขั้น ไม่มีตัวเลข ไม่มีบันได

- แสงที่เข้าตาเราไหลต่อเนื่อง
- เสียงที่เข้าหูเราไหลต่อเนื่อง
- ความร้อนที่ผิวเรารู้สึกไหลต่อเนื่อง
- การเคลื่อนไหวที่เรามองเห็นไหลต่อเนื่อง

นี่คือ **สัญญาณกายภาพ** และมันคือ “ภาษาแรก” ของโลกใบนี้

มนุษย์เกิดมาเข้าใจภาษาแบบนี้โดยธรรมชาติ เหมือนเด็กทารกที่ยังไม่รู้จักตัวเลข แต่รู้ว่าเสียงแม่อ่อนโยนกว่าเสียงคนอื่น

## **คอมพิวเตอร์รับรู้โลกแบบตัวเลข (Digital)**

แต่คอมพิวเตอร์… มันไม่มีตา ไม่มีหู ไม่มีผิว ไม่มีเวทนา ไม่มีอารมณ์

มันมีแต่ **ตัวเลข** มันเข้าใจโลกผ่าน “ค่าที่ถูกแปลงแล้ว” เท่านั้น

ดังนั้นเวลามนุษย์อยากให้คอมพิวเตอร์ช่วยแก้ปัญหา มนุษย์ต้อง “แปลภาษาโลกจริง” ให้เป็น “ภาษาตัวเลข” ก่อน

และนี่คือเหตุผลที่ IoT ต้องมีเซนเซอร์ เพราะเซนเซอร์คือ “อวัยวะรับรู้ของคอมพิวเตอร์”

##  **Host:**

งั้นมันก็เหมือนมนุษย์เลยนะครับ มนุษย์มีอวัยวะรับรู้ → สมองตีความ → ร่างกายตอบสนอง IoT ก็มีเซนเซอร์ → คอมพิวเตอร์ประมวลผล → อุปกรณ์ทำงาน

### **Guest:**

ใช่ครับ และมันคล้ายกันมากกว่าที่คิด เดี๋ยวผมจะค่อย ๆ ไล่ให้เห็นภาพแบบเป็นขั้นตอนนะครับ

# **อุปมา มนุษย์ = IoT หนึ่งระบบ**

| มนุษย์                                                       | IoT                                     |
| ------------------------------------------------------------ | --------------------------------------- |
| ตา หู จมูก ผิว                                               | เซนเซอร์                                |
| ความรู้สึกทันที เช่น ร้อน เจ็บ สว่าง มืด                     | สัญญาณแอนะลอก                           |
| การจำว่า “นี่คือเสียงเพื่อน”, “นี่คือแสงแดด”                 | ADC แปลงเป็นตัวเลข                      |
| การคิดต่อ เช่น “เพื่อนดูเครียดนะ”, <br>“แสงแบบนี้ถ่ายรูปสวย” | การประมวลผลของไมโครคอนโทรลเลอร์         |
| การรู้ว่า “โอเค ฉันเข้าใจแล้วว่ามันคืออะไร”                  | การตัดสินใจของไมโครคอนโทรลเลอร์         |
| การตอบสนองของร่างกาย เช่น เดิน วิ่ง หยิบของ                  | การสั่งงานอุปกรณ์ เช่น เปิดไฟ เปิดพัดลม |

มนุษย์รับรู้แบบต่อเนื่อง → IoT ก็รับรู้แบบต่อเนื่อง มนุษย์ตีความด้วยสมอง → IoT ตีความด้วยไมโครคอนโทรลเลอร์ มนุษย์ตอบสนองด้วยร่างกาย → IoT ตอบสนองด้วย actuator

 
- มนุษย์รู้สึกว่าฤดูหนาวปีนี้อากาศหนาวเกินไป ก่อนออกจากบ้านจึงใส่เสื้อผ้าให้หนาขึ้น
- คอมพิวเตอร์เห็นว่าอุณหภูมิที่เซนเซอร์อ่านได้ต่ำกว่า 25 องศาเซลเซียสจึงสั่งหยุดเครื่องทำความเย็น

มันคือ “ระบบเดียวกัน” ต่างกันแค่รูปแบบ

## **Host:**

งั้นสัญญาณกายภาพก็เหมือนประสบการณ์ที่มนุษย์รับรู้แบบต่อเนื่อง ส่วนสัญญาณแอนะลอกก็เหมือนความรู้สึกที่เกิดขึ้นทันที และไมโครคอนโทรลเลอร์ก็ทำงานเหมือนการตีความของสมอง?

### **Guest:**

ถูกต้องเลยครับ และเพื่อให้เห็นภาพชัดขึ้น ผมขอยกตัวอย่างที่เด็กทุกคนคุ้นเคยที่สุด…

# 📞 **โทรคุย = แอนะลอก / พิมพ์ไลน์ = ดิจิทัล**

### **Guest:**

เวลาเราโทรคุยกับเพื่อน เสียงพูดมันไหลต่อเนื่อง มีอารมณ์ มีน้ำเสียง มีความดังเบา ที่คนฟังรับรู้ และมีความรู้สึกร่วมตามไปด้วย ถ้าไม่อัดเสียงไว้ พูดจบก็หายไปเลย

นี่คือ **สัญญาณแอนะลอก** ในระบบไฟฟ้า คอมพิวเตอร์ไม่มีทางเข้าใจโดยตรง แต่ก็มีกระบวนการแปลงข้อมูลให้มันเข้าใจได้   

แต่ถ้าเราพิมพ์ไลน์ มันเป็นข้อความดิจิทัล เก็บได้ ส่งต่อได้ คอมพ์เข้าใจได้โดยตรง แต่… อารมณ์หายไปเยอะมาก

นี่คือ **สัญญาณดิจิทัล** เราพิมพ์ใส่คอมพิวเตอร์

### **Host:**

อ๋อ… งั้นมนุษย์เกิดมาเป็น “แอนะลอก” แต่คอมพิวเตอร์เกิดมาเป็น “ดิจิทัล” และ IoT คือการทำให้สองโลกนี้คุยกันรู้เรื่อง?

งั้นตั้งแต่แรกแล้ว **คีย์บอร์ดและหน้าจอ** คืออุปกรณ์ที่เราสื่อสารกับคอมพิวเตอร์สินะครับ


### **Guest:**

ใช่ครับ สองอย่างนั้นเราเรียกว่า **Terminal** ของคอมพิวเตอร์ ซึ่งมีมาตั้งแต่ยุคแรกๆ ของคอมพิวเตอร์เลย

คีย์บอร์ด จะส่งข้อมูลตัวเลขเข้าคอมพิวเตอร์โดยอาศัยความสามารถในการตีความของคน
เช่น ถ้าคิดถึงใคร เราอาจจะพิมพ์ข้อความ ส่งสติกเกอร์ หรือ อิโมจิ ซึ่งต้องผ่านกระบวนการคิดของคน ว่าจะใส่อะไรลงไป

แต่เชื่อไหมครับ มีอุปกรณ์ตัวหนึ่งที่ทำให้สองโลกนี้คุยกันรู้เรื่อง โดยเราไม่ต้องป้อนข้อมูลที่คีย์บอร์ดเลยก็คือ…

**ADC — ตัวแปลงสัญญาณแอนะลอกให้เป็นดิจิทัล**

คล้าย ๆ การแปลง “เสียงพูด” ให้เป็น “ข้อความพิมพ์” เพื่อให้คอมพิวเตอร์เข้าใจได้

### **Host:**

เหมือนเราพูดใส่มือถือแล้วมันพิมพ์ข้อความเองใช่ไหมครับ

### **Guest:**
ยังครับ ยังไม่ถึงขนาดนั้น การที่คอมพิวเตอร์พิใพ์แทนเรา ก็เริ่มจากการแปลงเสียงด้วย ADC แต่กระบวนมันยาวกว่านั้น มันต้องใช้ระบบจดจำและวิเคราะห์เสียงพูดเพื่อดึงเอาข้อความมาป้อนใส่แป้นพิมพ์

### **Host:**
โอ้โห ถ้างั้นตอนที่เราพูดใส่มือถือนี่ มันทำงานซับซ้อนมากเลยนะครับ 

### **Guest:**

ใช่ครับ ตอนหน้าเราจะเข้าเรื่องสำคัญที่สุดของ IoT เลย ว่า **ADC ทำงานยังไง**, ทำไมต้องมี, และมันแปลโลกจริงให้คอมพิวเตอร์เข้าใจได้ยังไง แต่ยังไม่ถึงกับการถอดคำพูดเป็นตัวพิมพ์นะครับ อันนั้นต้องอีกเรื่องยาวๆ เลยครับ

### **Host:**

โอเคครับ งั้นตอนที่ 3 ขอจบไว้เท่านี้ แล้วเจอกันตอนหน้าในหัวข้อ **ADC: ตัวแปลงเวทนาให้เป็นความคิดของคอมพิวเตอร์**

**สวัสดีครับ**
