# **ตอนที่ 6 — The Interrupt & Runtime Presence**

## _“วิญญาณ (Vinnana): The System Awareness”_

### **Host:**

สวัสดีครับทุกคน ยินดีต้อนรับเข้าสู่ตอนที่ 6 ของรายการ “สนทนาภาษาเซนเซอร์” ตอนที่แล้วเราเพิ่งคุยกันเรื่อง **สังขาร** หรือ **Processing Logic** ว่ามันคือส่วนที่คอมพิวเตอร์ “คิด” และ “ตัดสินใจ” ตามลอจิกที่ถูกเขียนไว้

แต่การคิดอย่างเดียวไม่พอครับ เพราะต่อให้ลอจิกดีแค่ไหน ถ้าระบบ “ไม่รับรู้” ว่ามีเหตุการณ์เกิดขึ้น มันก็จะไม่ทำงานอยู่ดี

วันนี้เราจะเข้าสู่ชั้นสุดท้ายของระบบขันธ์ ๕ นั่นคือ **วิญญาณ (Vinnana)** หรือในภาษาวิศวกรรมก็คือ…

> **Interrupt & Runtime Awareness** ระบบที่ “รู้ตัว” ว่ามี Event เกิดขึ้น

### **Guest:**

ใช่ครับ Host หลายคนพอได้ยินคำว่า “วิญญาณ” ก็จะคิดถึงอะไรล่องลอยหลังความตาย แต่ในทางสถาปัตยกรรมของจิต และในทางคอมพิวเตอร์ วิญญาณคือสิ่งที่ทำงานอยู่ทุกวินาทีที่ระบบเปิดอยู่

มันคือ **สภาวะการรับรู้** หรือถ้าเป็นสาย Embedded อย่างเรา ก็เรียกว่า…

> **Interrupt Service Routine — ISR**

ถ้าไม่มี ISR ต่อให้มีแรงดันไฟฟ้าวิ่งเข้าขา Pin ต่อให้เซนเซอร์ส่งสัญญาณเข้ามา ต่อให้ ADC แปลงเป็นตัวเลขแล้ว ระบบก็จะ “ไม่รับรู้” อะไรเลย

### **Host:**

งั้นวิญญาณก็คือสิ่งที่ “เด้งขึ้นมา” เฉพาะตอนที่มีเหตุการณ์?

### **Guest:**

ถูกต้องครับ วิญญาณไม่ใช่สิ่งที่แช่อยู่ตลอดเวลา มันเกิดขึ้นเฉพาะเมื่อมี Event มากระทบ

เหมือนจักขุวิญญาณที่เกิดเฉพาะตอนมีแสงเข้าตา เหมือนโสตวิญญาณที่เกิดเฉพาะตอนมีเสียงเข้าหู

พอ Event หายไป → วิญญาณก็ดับลง มันเป็น **Event-driven Awareness**

### **Host:**

งั้นถ้าเราเปิดตาอยู่ แต่ใจลอยไปที่อื่น แสงเข้าตา (รูป) มีความรู้สึกทางตา (เวทนา) มีภาพปรากฏ (สัญญา) แต่เรา “ไม่เห็น” ว่ามันคืออะไร

นี่คือสภาวะที่ “จักขุวิญญาณไม่เกิด” ใช่ไหมครับ?

### **Guest:**

ใช่เลยครับ มันเหมือน CPU ที่ติดอยู่ใน Infinite Loop ของสังขาร จนลืมเช็ค Interrupt จากภายนอก

เครื่องไม่ได้ดับ แต่เครื่อง “ไม่รับรู้”

นี่คือสภาวะที่มนุษย์เรียกว่า “ใจลอย” แต่ในทาง Embedded คือ **ISR ไม่ถูก Trigger**

### **Host:**

หลายคนคิดว่า วิญญาณคือเจ้าของร่าง แล้ววิญญาณนี่…มันเป็นเจ้าของเครื่องไหมครับ?

### **Guest:**

นี่แหละครับจุดสำคัญที่สุดของตอนนี้

> **วิญญาณไม่ใช่เจ้าของเครื่อง** มันเป็นเพียง “สภาวะการรู้” ที่เกิดขึ้นตามเหตุปัจจัย

มันไม่ใช่ CPU ไม่ใช่โปรแกรมเมอร์ ไม่ใช่ตัวตนถาวร

มันคือ **Data Bus** คือการไหลของข้อมูลระหว่าง Input → Process

ถ้าไม่มี Input → วิญญาณไม่เกิด ถ้ามีการรับรู้โดยไม่มี Input → นั่นคือ “จิตปรุงแต่ง” หรือ Noise ในระบบ


### **Host:**
 ถ้าผมเริ่มงง แสดงว่านักศึกษางงกว่านะครับ 555
### **Guest:**

สังขารกับวิญญาณมันจะทำงานคู่กันเสมอ เมื่อมีเหตุการณ์ภายนอก

- **วิญญาณ = การรับรู้ว่าเหตุการณ์เกิดขึ้น** (Interrupt)
- **สังขาร = การตัดสินใจว่าจะทำอะไรกับเหตุการณ์นั้น** (Processing Logic)

มันคือคู่ที่ทำงานแบบ:

**Event → Awareness → Logic → Action**

เหมือนใน MCU:
1. ISR เด้งขึ้นมา (วิญญาณ)
2. Logic ตัดสินใจ (สังขาร)
3. ส่งผลลัพธ์ออกไป (ร่างกาย)
### **Host:**
ถ้างั้น สังขารก็คือ process หรือ loop ที่รอประมวลผลหรือครับ
### **Guest:**
ยังไม่ถูกครับ

สังขารไม่ใช่ `while(true)` ที่รอเหตุการณ์ 
ไม่ใช่ loop ที่นั่งรอ input 
ไม่ใช่ thread ที่ idle อยู่เฉย ๆ

แต่สังขารคือ **ฟังก์ชันที่ถูกเรียกขึ้นมาเพื่อ “คิด” และ “ตัดสินใจ”** ทันทีที่วิญญาณ (ISR) แจ้งว่า “มีเหตุการณ์เข้ามาแล้วนะ!”
เหมือนใน MCU

```
ISR() {
    processLogic();   // ← นี่แหละ สังขาร
}
```

ดังนั้นสังขารคือ **Processing Logic** 
ไม่ใช่ **Waiting Loop**



### **Host:**

โอเค พยายามจะเข้าใจครับ 

งั้นเราไปต่อ 

วิญญาณก็เกิดดับตาม Clock Speed ของระบบ?

### **Guest:**

ใช่ครับ มันเกิดดับเร็วมาก ระดับไมโครวินาที นาโนวินาที

มันไวจนระบบที่ “ขาดสติ” เข้าใจผิดว่า การรับรู้นี้มันนิ่ง เป็นเนื้อเดียวกัน แล้วเหมาว่าเป็น “เรา” ที่กำลังรับรู้อยู่ตลอดเวลา

แต่จริง ๆ แล้วมันคือ **Interrupt ที่เกิดเป็นช่วง ๆ เท่านั้น**

### **Host:**

แล้ววิญญาณมีหลายช่องทางเหมือน Port ของ MCU ใช่ไหมครับ?

### **Guest:**

ใช่ครับ ในพุทธมีวิญญาณ ๖ ช่องทาง เหมือน Multi-Port Interface ของ MCU
- จักขุวิญญาณ → Port ตา
- โสตวิญญาณ → Port หู
- ฆานวิญญาณ → Port จมูก
- ชิวหาวิญญาณ → Port ลิ้น
- กายวิญญาณ → Port ผิว
- มโนวิญญาณ → Port ใจ (Internal Event)
แต่ละ Port ทำงานอิสระ เกิดดับตาม Event ไม่มี “ตัวรู้หนึ่งเดียว” ที่แช่อยู่ตลอดเวลา

### **Host:**
โอเคครับ ยิ่งฟังยิ่งปวดหัว

งั้นสรุปตอนนี้ได้ว่า…

### **Guest :**

> **วิญญาณ = Interrupt ที่ทำให้ข้อมูลปรากฏตัวขึ้นในระบบ**
> 
> ถ้าไม่มีวิญญาณ → ระบบไม่รับรู้ 
> ถ้าวิญญาณผิดพลาด → ระบบรับรู้ผิด 
> วิญญาณไม่ใช่เจ้าของเครื่อง → มันเป็นเพียง Event-driven Awareness


บางทีเราออกแบบระบบมาดีทั้งหมด แต่มีบางอย่างมาทำให้ระบบไม่ตอบสนอง 
เช่น
- แม่เรียก แต่เราไม่ตอบ เพราะใจลอย → ISR ไม่เด้ง
- ขับรถแล้วไม่เห็นไฟแดง เพราะคิดมาก → วิญญาณไม่เกิด ฟังก์ชันเหยียบเบรคไม่ถูกเรียก (ไม่ใช่ไม่ทำงาน หรือทำงานผิด แต่ไม่ถูกเรียกใช้)
- เซนเซอร์ควันไฟส่งสัญญาณ แต่ ISR ไม่เด้ง → ไฟไหม้บ้าน (อาจจะต่อเข้าผิดพอร์ต)
- ปุ่มกดมี Bounce แต่ ISR รับผิด → ระบบสั่งงานผิด
- Interrupt ถูก Mask ไว้ → ระบบไม่รับรู้เหตุการณ์สำคัญ

### **Host:**

ตอนนี้เราครบทั้ง 5 ชั้นของระบบขันธ์ ๕ แล้วนะครับ ตั้งแต่รูป เวทนา สัญญา สังขาร และวิญญาณ 
ซึ่งมันสอดคล้องกับระบบ IoT แบบน่าทึ่ง

ตอนหน้าเราจะเข้าสู่ **บทสรุปใหญ่ที่สุดของซีรีส์นี้** ว่าเมื่อเรามองระบบทั้งหมดแบบนี้แล้ว 
เราจะเข้าใจ “ชีวิต” และ “ระบบ IoT” แบบใหม่ได้อย่างไร
